# Export System Design

---

## Overview

The export system generates 9 files and bundles them into a ZIP download. All generation is client-side. No server involved.

**Entry point:** `triggerZipDownload(state: WizardState): Promise<void>`

Called from the Export step's Download button.

---

## File List

| File | Generator | Content |
|---|---|---|
| `PROJECT_GUIDELINES.md` | `generators/guidelines.ts` | Full stack decisions in readable Markdown |
| `AI_CONTEXT.md` | `generators/aiContext.ts` | Structured prompt context for AI assistants |
| `WHY.md` | `generators/why.ts` | Rationale for each major decision |
| `README.md` | `generators/readme.ts` | Project overview with tech stack badges |
| `project-config.json` | `generators/config.ts` | Machine-readable full config as JSON |
| `tokens.css` | `generators/tokensCss.ts` | CSS custom properties for design tokens |
| `tokens.tailwind.json` | `generators/tokensTailwind.ts` | Flat JSON for Tailwind 4 `@theme` block |
| `.gitignore` | `generators/gitignore.ts` | .gitignore based on selected framework |
| `apply-blueprint.js` | `generators/applyBlueprint.ts` | Node.js installer — scaffolds project, installs deps, creates folders, injects tokens |

## Generator Pattern

Each generator is a pure function:

```typescript
// src/export/generators/guidelines.ts
export function generateGuidelines(state: WizardState): string {
  const { project, architecture, standards } = state;
  return `# Project Guidelines

Generated by FrontForge on ${new Date().toLocaleDateString()}.

## Application Stack

- **Framework:** ${frameworkLabel[project.framework] || 'Not specified'}
- **Language:** ${project.language === 'typescript' ? 'TypeScript' : 'JavaScript'}
- **Package Manager:** ${project.packageManager}

## State Management

${project.stateManagement.length
  ? project.stateManagement.map((s) => `- ${stateLabel[s]}`).join('\n')
  : '- No global state management (use server state / Context)'}

## Styling

${project.styling.map((s) => `- ${stylingLabel[s]}`).join('\n')}

## Architecture

- **Folder Strategy:** ${architectureLabel[architecture.folderStrategy]}
- **Naming Convention:** ${architecture.namingConvention}
- **Path Aliases:** ${architecture.pathAliases ? `\`${architecture.aliasRoot}/*\`` : 'None'}
- **Barrel Files:** ${architecture.barrelFiles}
- **Environment Variables:** ${architecture.envStrategy === 'schema-validated' ? 'Schema-validated (t3-env)' : 'dotenv only'}

## Testing

- **Unit Tests:** ${standards.testingUnit === 'none' ? 'None' : standards.testingUnit}
- **E2E Tests:** ${standards.testingE2E === 'none' ? 'None' : standards.testingE2E}

## Code Standards

- **Error Handling:** ${standards.errorHandling}
- **Git Strategy:** ${standards.gitStrategy === 'conventional-commits' ? 'Conventional Commits' : 'Custom'}
- **Linting:** ${standards.linting.join(', ') || 'None configured'}
`;
}
```

## AI_CONTEXT.md Format

Optimized for pasting into AI assistant context windows (Claude, GPT, Copilot):

```
# AI Context — [Project Name]

This document defines the technical decisions for this project.
When generating code, follow these decisions strictly.

## Stack
Framework: Next.js (App Router)
Language: TypeScript (strict mode)
Package Manager: pnpm
State: Zustand
...

## Architecture Rules
- Folder structure: feature-based
- Path alias: @/* maps to src/*
- Barrel files: feature-level only
...

## Do Not Use
- Redux (use Zustand)
- Formik (use React Hook Form)
- moment.js (use dayjs)
```

## WHY.md Approach

Map each option value to a pre-written rationale paragraph:

```typescript
const stateRationale: Record<string, string> = {
  'zustand': 'Zustand was chosen for its minimal API surface (~1KB), zero boilerplate, and excellent TypeScript support. It handles both simple and complex global state without Redux\\'s ceremony.',
  'redux-toolkit': 'Redux Toolkit was chosen for its standardized patterns, powerful DevTools integration, and strong ecosystem — particularly valuable for large teams and complex state machines.',
  'context-only': 'React Context was chosen to avoid adding a state library. This works well when global state is minimal and updates are infrequent (since Context re-renders all consumers).',
};
```

WHY.md assembles these paragraphs per decision. This requires writing ~60 rationale paragraphs — budget time for this copy.

## project-config.json

Full state serialized to clean JSON. This is the machine-readable single source of truth. Schema versioned.

```json
{
  "schemaVersion": 1,
  "generatedAt": "2026-06-18T...",
  "project": {
    "name": "my-app",
    "framework": "next",
    "language": "typescript",
    "stateManagement": ["zustand"],
    ...
  },
  "architecture": { ... },
  "designSystem": { ... },
  "standards": { ... },
  "uxPatterns": { ... }
}
```

This file is also the **import format** — a future "Import Blueprint" feature reads this JSON and restores all decisions.

## ZIP Bundler

```typescript
// src/export/zip.ts
export async function triggerZipDownload(state: WizardState): Promise<void> {
  // Lazy load — only executes on export click, ~90KB not on initial load
  const JSZip = (await import('jszip')).default;
  const zip = new JSZip();

  zip.file('PROJECT_GUIDELINES.md', generateGuidelines(state));
  zip.file('AI_CONTEXT.md', generateAIContext(state));
  zip.file('WHY.md', generateWhy(state));
  zip.file('README.md', generateReadme(state));
  zip.file('project-config.json', JSON.stringify(generateConfig(state), null, 2));
  zip.file('tokens.css', generateTokensCss(state));
  zip.file('tokens.tailwind.json', JSON.stringify(generateTokensTailwind(state), null, 2));
  zip.file('.gitignore', generateGitignore(state));
  zip.file('apply-blueprint.js', generateApplyBlueprint(state)); // added in changes.md

  const blob = await zip.generateAsync({ type: 'blob' });
  const url = URL.createObjectURL(blob);
  const a = document.createElement('a');
  a.href = url;
  a.download = `${slugify(state.project.projectName || 'project')}-blueprint.zip`;
  document.body.appendChild(a);
  a.click();
  document.body.removeChild(a);
  URL.revokeObjectURL(url);
}
```

**Do not use `file-saver`** — `URL.createObjectURL` + programmatic click works in all modern browsers.

## apply-blueprint.js

The installer script runs in Node.js with zero dependencies (uses built-in `fs`, `path`, `child_process`, `readline`).

```
node apply-blueprint.js           # interactive — shows summary, prompts Y/n, executes
node apply-blueprint.js --dry-run # prints all intended actions, creates nothing
```

**Steps:** scaffold project → install prod deps → install dev deps → create folder structure → write placeholder files (api.ts, constants.ts, queryClient.ts, store/index.ts) → copy docs to /docs → inject tokens.css → update tsconfig.json paths → write biome.json / .prettierrc → init husky.

**Philosophy:** every write calls `writeIfMissing()` (checks `fs.existsSync` first). Config additions use `appendIfMissing()`. Never overwrites existing files.

## Export Step UX

The Export step shows a checklist of all 9 files with:
- File name
- Brief description
- Line count estimate (computed from generated string length)
- A preview of the first ~10 lines (expandable)

One "Download All (ZIP)" button. No individual file downloads in MVP (clipboard copy = v1.1).
